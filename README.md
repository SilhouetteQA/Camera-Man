# 姿态监控系统 (Camera Man)

USB 摄像头实时坐姿监控，检测驼背/前倾/歪头/久坐/手机使用，Windows 桌面弹窗 + 提示音提醒。

## 快速开始

```bash
# 1. 首次使用：安装依赖（只需执行一次）
pip install opencv-python mediapipe pystray Pillow

# 2. 进入项目目录，启动
cd "D:\AI project\camera man"
python main.py

# 3. 系统托盘出现蓝色图标，右键 → 开始监控
```

首次运行 MediaPipe 会自动下载姿态模型（~5MB），等待几秒即可。后续使用直接执行第 2 步。

## 功能

| 功能 | 说明 | 阈值 |
|------|------|------|
| 驼背检测 | 耳-肩-髋连线偏离垂直角度 | > 4° |
| 前倾检测 | 上半身与垂直方向夹角 | > 15° |
| 歪头检测 | 双耳连线与水平线夹角 | > 6° |
| 久坐提醒 | 画面中持续检测到人体累计 | 60 分钟 |
| 手机使用 | 头部下倾看手机累计时间 | 10 分钟 |
| MinimaX M3 验证 | 多模态模型二次验证（可选） | 托盘手动开关 |

### 去抖与冷却

- 连续 3 帧（15 秒）确认不良姿态后才触发告警
- 同类型告警冷却 5 分钟，期间不重复弹窗
- 画面无人自动暂停，人回来自动恢复

### 手机使用计时

- 检测逻辑：鼻子与肩膀的垂直距离缩小（低头看手机时鼻子下降接近肩膀）
- 累计满 10 分钟弹窗提醒
- 短暂抬头（2 分钟内）不重置计时，抬头超过 2 分钟才清零
- 提示语："你已经看手机超过 10 分钟，放下手机休息一下吧"

## 托盘操作

```
右键托盘图标 →
  ├─ 开始监控             启动摄像头和姿态检测
  ├─ 暂停                 停止监控
  ├─ MinimaX 视觉验证     勾选启用 M3 二次验证（需环境变量 minimax_api）
  └─ 退出                 关闭程序
```

## MinimaX M3 视觉验证

1. 设置系统环境变量 `minimax_api` 为你的 API Key
2. 托盘菜单项 "MinimaX 视觉验证" 变为可点击
3. 勾选后启用，M3 对本地 MediaPipe 的检测结果做二次确认
4. M3 只做减法（过滤误报），不做加法（不引入新检测）
5. API 调用间隔 ≥ 30 秒，单次超时 5 秒，失败不影响主流程

## 判定逻辑

```
摄像头 5s/帧 → MediaPipe 提取 33 关键点 → 规则计算角度

驼背: 耳中点-肩中点-髋中点 三点连线偏离 180° 的角度
前倾: 鼻子-肩中点 连线与垂直方向夹角
歪头: 左耳-右耳 连线与水平线夹角
手机: 鼻子与肩膀的垂直距离缩小（头部下倾看手机）

逐帧判定 → 去抖(连续3帧确认) → 冷却检查 → MessageBox 弹窗 + 提示音
久坐: 每 60 秒累计 1 分钟 → 满 60 分钟弹窗
手机: 每 60 秒累计 1 分钟 → 满 10 分钟弹窗（2分钟抬头宽容期）
```

## 模块

```
main.py          主循环 + 去抖状态机 + 久坐/手机计时
camera.py        OpenCV DSHOW 摄像头管理
analyzer.py      MediaPipe Pose 33 关键点 + 角度判定规则
alerter.py       Windows MessageBox 弹窗 + 提示音 + 冷却
storage.py       SQLite 事件记录 + 每日统计
config.py        AppConfig dataclass 默认配置
vision_client.py Minimax M3 API 客户端
tray_ui.py       系统托盘菜单
```

## 数据

SQLite 存储在 `data/posture.db`，两张表：
- `events` — 事件明细（时间、类型、严重度、置信度）
- `daily_stats` — 每日汇总（久坐分钟、各类告警次数）

## 开发记录

通过 superpowers 标准流程开发：spec → plan → TDD → SDD（subagent-driven-development），7 个任务模块，40 个单元测试。

阈值经真实摄像头数据校准：摄像头俯拍角度导致 MediaPipe 关键点接近垂直，实际姿势偏差在画面中仅体现 2-6°，因此阈值设置较低以避免漏检。
