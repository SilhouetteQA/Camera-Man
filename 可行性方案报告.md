# 姿态监控系统 — 可行性方案报告

> 2026-06-25 | 版本 v1.0

---

## 1. 项目概述

通过 USB 摄像头监控用户在电脑前的坐姿、久坐时间和手机使用行为，在检测到不良姿态或超时久坐时通过桌面弹窗和声音提醒用户。系统以纯 PC 单机运行，后续可扩展手机端推送。

### 1.1 核心目标

| 目标 | 描述 |
|------|------|
| 坐姿检测 | 识别驼背、前倾、歪头等不良姿态 |
| 久坐提醒 | 累计坐姿时长，超阈值触发提醒 |
| 手机使用检测 | 识别用户是否在玩手机 |
| 即时反馈 | Windows 原生 Toast 通知 + 声音提示 |

### 1.2 二期扩展方向

- 手机端 App 推送（局域网 WebSocket）
- 多模态大模型复杂场景理解（云端兜底）
- 每日/每周姿态健康报告
- 多摄像头多角度覆盖

## 3. 信息处理方法

### 3.1 整体流程

```
USB摄像头 → OpenCV采集帧 → 定时采样(每3-5秒) → 姿态分析 → 状态判定 → 告警触发
                                      ↓
                                手机检测(并行)
```

### 3.2 姿态估计 — MediaPipe Pose

- **算法：** Google MediaPipe Pose (BlazePose)
- **输出：** 33 个 3D 人体关键点（x, y, z, visibility）
- **性能：** CPU 即可实时推理（~30fps on i5），无需 GPU
- **关键点利用：**

| 检测项 | 利用关键点 | 判定逻辑 |
|--------|-----------|----------|
| 驼背 | 肩膀(11,12) + 髋部(23,24) + 耳朵(7,8) | 耳-肩-髋连线曲率超过阈值 |
| 前倾 | 鼻子(0) + 肩膀中点 + 髋部中点 | 上半身与垂直方向夹角 > 15° |
| 歪头 | 耳朵(7,8) + 肩膀(11,12) | 耳-肩连线与水平线夹角 > 10° |
| 久坐 | 髋部(23,24) 可见性 | 持续检测到人体视为在座，累计时长 |

### 3.3 手机使用检测

**方案：** 结合以下信号综合判断：

1. **手部位置检测** — MediaPipe Pose 手腕关键点(15,16) 接近头部区域
2. **手机目标检测** — 轻量 YOLOv8-nano 检测画面中是否出现手机（cell phone 类别）
3. **头部下倾** — 鼻子(0) 与肩膀中点连线向下倾斜

三者叠加可判定为"正在玩手机"。

**替代方案：** 也可只用手部-头部距离 + 头部下倾两个姿态信号，不引入 YOLO，进一步降低算力开销。准确率略有下降但足以满足日常提醒需求。

### 3.4 云端多模态兜底（二期）

当本地算法连续多帧置信度低于阈值时，可选择性调用：

- OpenAI GPT-4V / Claude Vision / Gemini Vision API
- 发送当前帧图片 + 上下文提示词
- 获取场景理解结果作为二次判断
- 单次调用成本约 $0.01-0.03，正常使用下日均调用 < 50 次

### 3.5 防误报策略

- **状态去抖：** 连续 N 帧（如 3 帧）判定为同一不良状态才触发告警
- **告警冷却：** 同一类型告警间隔不少于 T 分钟（如坐姿提醒间隔 5 分钟）
- **缺席检测：** 连续 M 帧未检测到人体则暂停所有告警（人不在座位）
- **工作时间窗口：** 可配置仅在特定时段运行（如 9:00-22:00）

## 4. 技术栈

### 4.1 主控语言：Python 3.10+

| 层级 | 库/框架 | 用途 |
|------|---------|------|
| 图像采集 | OpenCV (`opencv-python`) | 摄像头读取、帧预处理 |
| 姿态估计 | MediaPipe (`mediapipe`) | 33 关键点姿态推理 |
| 物体检测 | Ultralytics YOLO (`ultralytics`) | 手机目标检测（可选） |
| 桌面通知 | `win10toast` 或 `plyer` | Windows Toast 弹窗 |
| 声音播放 | `pygame` | 播放本地提示音 |
| 系统托盘 | `pystray` | 托盘图标 + 右键菜单 |
| 数据存储 | `sqlite3` (标准库) | 姿态事件 + 久坐统计 |
| 定时调度 | `threading.Timer` 或 `schedule` | 采样间隔控制 |
| 配置管理 | `tomllib` (Python 3.11+) 或 `configparser` | 用户设置持久化 |

### 4.2 云端 API（二期，可选）

| 服务 | SDK | 用途 |
|------|-----|------|

| Gemini Vision | `google-generativeai` | Google 多模态兜底 |

### 4.3 运行环境要求

## 5. 系统架构

### 5.1 模块划分

```
camera-man/
├── main.py              # 入口：启动托盘、调度器
├── camera.py            # OpenCV 摄像头管理 + 帧采集
├── analyzer.py          # 姿态分析 + 手机检测 + 状态判定
├── alerter.py           # 弹窗 + 声音提醒 + 防骚扰控制
├── storage.py           # SQLite 事件记录 + 统计查询
├── config.py            # 配置加载/保存
├── tray_ui.py           # 系统托盘 UI
├── sounds/
│   └── alert.wav        # 提示音文件
└── data/
    └── posture.db        # SQLite 数据库（自动创建）
```

### 5.2 数据流

```
┌─────────┐    帧     ┌──────────┐   关键点/检测结果   ┌──────────┐
│ camera  │ ────────→ │ analyzer │ ────────────────→ │ 状态机   │
│ OpenCV  │           │ MediaPipe│                    │ 去抖判定 │
└─────────┘           └──────────┘                    └────┬─────┘
                                                          │
                                      ┌───────────────────┤
                                      ↓                   ↓
                                 ┌─────────┐        ┌──────────┐
                                 │ alerter │        │ storage  │
                                 │ 弹窗声音 │        │ SQLite   │
                                 └─────────┘        └──────────┘
```

### 5.3 主循环伪代码

```python
while running:
    frame = camera.capture()
    if sampler.should_sample():          # 定时采样
        pose = analyzer.estimate_pose(frame)
        phone = analyzer.detect_phone(frame)
        events = state_machine.update(pose, phone)
        for event in events:
            alerter.fire(event)
            storage.record(event)
    sleep(0.1)                           # 释放 CPU
```

## 6. 可行性评估

### 6.1 技术可行性：高

- MediaPipe Pose 在 CPU 上成熟稳定，Python SDK 完善
- OpenCV 兼容所有主流 USB 摄像头
- 所有组件均为开源，零授权费用
- 核心依赖仅 3 个包：opencv-python、mediapipe、pystray

### 6.2 精度评估

| 检测项 | 预期准确率 | 主要误差来源 |
|--------|-----------|-------------|
| 是否在座 | > 95% | 遮挡、暗光 |
| 驼背 | ~85% | 宽松衣物、坐姿多变 |
| 前倾 | ~80% | 摄像头角度偏移 |
| 歪头 | ~85% | 转头与歪头易混淆 |
| 玩手机 | ~75% | 手部遮挡、判定边界模糊 |

### 6.3 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 光照不足/逆光 | 姿态关键点丢失 | 提醒用户调整室内光线；可加补光灯 |
| 摄像头被遮挡 | 全部检测失效 | 连续丢失关键点时静默，不误报 |
| 衣物过于宽松 | 姿态关键点偏移 | MediaPipe 对衣着鲁棒性较好，大部分情况可用 |
| 多人在画面中 | 误检测他人 | 仅取画面中最大的人体框作为目标 |
| CPU 占用过高 | 影响正常工作 | 定时采样（3-5秒间隔）使 CPU 占用 < 5% |

## 8. 总结

本方案以 **Python + MediaPipe + OpenCV** 为核心技术栈，在 Windows 平台实现坐姿监控和久坐提醒功能。技术路线成熟、依赖少、开发周期短，MVP 预计可在 1-2 周内完成。

核心优势：
- 纯本地处理，隐私安全
- 零费用，全开源技术栈
- CPU 即可运行，无需 GPU
- 架构简洁，易于扩展

建议优先推进 Phase 1 MVP，快速验证姿态检测在实际场景中的精度，再根据体验反馈调整后续方向。
